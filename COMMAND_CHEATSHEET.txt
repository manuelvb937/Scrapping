Social Listening Pipeline Command Cheatsheet
===========================================

Run these commands from the project root:

  C:\Users\MANUE\Desktop\Scrapping


Current Analysis Pipeline
-------------------------

The analyze command now runs:

  1. preprocess posts for clustering
  2. transformer sentiment per post -> Sentiment_Transformer
  3. embeddings
  4. UMAP + HDBSCAN clustering
  5. c-TF-IDF keywords per cluster
  6. LLM batches inside each cluster -> Sentiment_LLM + up to 3 raw topics
  7. algorithmic topic consolidation inside each cluster
  8. second LLM consolidation -> compact canonical topic taxonomy
  9. canonical topics attached back to posts
  10. final LLM marketing call -> interpretation, risk, opportunity, mini report


Basic Full Pipeline
-------------------

Scrape posts:

  python cli.py scrape --keywords "おっさんずラブ"

Preprocess the latest scraped file:

  python cli.py preprocess

Analyze the latest processed file:

  python cli.py analyze

Analyze with free-tier API protections:

  python cli.py analyze --free

Generate markdown/html/csv report:

  python cli.py report


Specific File: Gemini Free-Tier Analysis
----------------------------------------

Use this when you already have a processed JSONL file.

PowerShell:

  cd C:\Users\MANUE\Desktop\Scrapping
  $env:LLM_PROVIDER="gemini"
  $env:GEMINI_API_KEY="your_gemini_key_here"
  $env:GEMINI_MODEL="gemini-2.5-flash"
  $env:TOPIC_LABELING_MODEL="gemini-2.5-flash"
  python cli.py analyze --free --input "data/processed/YOUR_FILE_processed.jsonl"

If the file is in the project root instead of data/processed:

  python cli.py analyze --free --input ".\YOUR_FILE_processed.jsonl"

Outputs are written to:

  data/reports/clusters.json
  data/reports/analysis.json
  data/reports/structured_output.json


Free-Tier Defaults
------------------

The --free flag sets these defaults only if you have not already set them:

  SENTIMENT_METHOD=transformer
  ENABLE_LLM_SENTIMENT_FALLBACK=false
  HDBSCAN_MIN_CLUSTER_SIZE=6
  HDBSCAN_MIN_SAMPLES=2
  HDBSCAN_CLUSTER_SELECTION_METHOD=eom
  POST_TOPIC_LLM_BATCH_SIZE=15
  LLM_POST_BATCH_SIZE=15
  LLM_TOPIC_CONSOLIDATION_MAX_TOPICS=6
  LLM_TOPIC_CONSOLIDATION_MAX_INPUT_TOPICS=40
  LLM_POST_TOPIC_MAX_OUTPUT_TOKENS=4096
  LLM_MARKETING_MAX_OUTPUT_TOKENS=4096
  LLM_MARKETING_MAX_CLUSTERS=60
  GEMINI_FREE_TIER_LIMITING=true
  GEMINI_REQUESTS_PER_MINUTE=10
  GEMINI_TOKENS_PER_MINUTE=250000
  GEMINI_REQUESTS_PER_DAY=250
  GEMINI_MIN_SECONDS_BETWEEN_REQUESTS=6
  GEMINI_THINKING_BUDGET=0
  LLM_TEMPERATURE=0
  LLM_TOP_P=1

Meaning:

  - The transformer model analyzes every post locally.
  - HDBSCAN uses larger, more stable clusters by default.
  - The LLM receives cluster-local batches of 15 posts.
  - The LLM returns only Sentiment_LLM plus concise topics, no rationales.
  - A second LLM pass consolidates preliminary topics into up to 6 final topics per cluster.
  - One final LLM call creates marketing interpretation and mini report.
  - Gemini calls are paced by a shared request/token/day limiter.


No LLM Key Mode
---------------

If OPENAI_API_KEY and GEMINI_API_KEY are not set, analyze still completes:

  Remove-Item Env:OPENAI_API_KEY -ErrorAction SilentlyContinue
  Remove-Item Env:GEMINI_API_KEY -ErrorAction SilentlyContinue
  python cli.py analyze --input "data/processed/YOUR_FILE_processed.jsonl"

In this mode:

  - Sentiment_Transformer still runs.
  - Sentiment_LLM falls back to the transformer label.
  - Topics fall back to c-TF-IDF keywords.
  - Marketing fields use deterministic fallback rules.
